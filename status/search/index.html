Search API status

Context: the primary website www.onetoo.eu is a static trust-root. The actual search API (historically on search.onetoo.eu) may be deployed as a separate Cloudflare Worker / service.

What this means

  • If search.onetoo.eu returns 404, it usually means the Worker route is not currently attached, is under maintenance, or the subdomain is pointing elsewhere.
  • This repository still publishes search-related trust signals (OpenAPI, schemas, signed manifests) that remain valid even while the runtime API is being rebuilt.

Where to look next

  • Current public API docs: /openapi.json
  • Status (machine-readable): /status.json
  • Incidents: /incidents/

If you operate the API yourself, consider re-attaching the Worker route for search.onetoo.eu and publishing a signed note in /incidents/.
